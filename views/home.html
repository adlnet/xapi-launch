xAPI Launch

This system allows users to record content at any URL, and track their experiences with that content. User information such as a name and an email address are provide to the content so it can log interactions. The server sets up a single use xAPI proxy, and provides this proxy location to the content. Each launch of the content is associated with a one-time-use token, which activates the proxy endpoint and establishes a session for the content. Anyone may submit data to this endpoint given that they have access to the launch token and the session key. However, once the launch token has been exchanged for the launch information, only clients with the proper session token can request the user data or post to the xAPI endpoint. The link submitter can enter an optional RSA Public Key for the content. If provided, the API data will be encrypted before being returned to the user. The user therefore may not know the API endpoints.

All statements which are logged via the one-time-use proxy endpoint are automatically associated with a context activitiy to identify the content from which it came, as well as a context activity to identify the session of the content during which it was posted. In this way, a student can see the result for each launch of the content, and a content owner can see the results of each attempt on the course.

The static content file located in the /public folder implements the most basic possible interaction. It uses a library to manage to token exchange round-trip, and specifies that the library should consider the attempt complete when the page unloads. Closing the attempt is the applications responsibility.
